# わかったこと 
### 正規化についてのまとめ
+ 正規化の三つのポイント
    + 正規化とは、更新時の不都合/不整合を排除するために行う
    + 正規化は、従属性を見抜くことで可能になる
    + 正規形はいつでも非正規形に戻せる
+ 正規化は常にするべきか？
    + 第3正規形までは原則として行う
    + 関連エンティティが存在する場合は、関連とエンティティが1対1に対応するよう注意する
+ 正規化を行う利点
    + 1.データの冗長性が排除され、更新時の不整合を防止できる
    + 2.テーブルの持つ意味が明確になり、開発者が理解しやすい
+ 正規化を行う欠点
    + テーブルの数が増えるため、SQL文で結合を多用することになり、パフォーマンスが悪化する
